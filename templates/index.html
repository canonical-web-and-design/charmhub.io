Pure Python Operators

Python rocks for DevSecOps

Take “infrastructure as code” to the next level.

Control all your K8s workloads with Python, a high-level language with a huge community. Reuse open-source Python libraries for all your integration. Get standard libraries for standard workloads such as MySQL, Postgres, Etcd, Vault, MongoDB, Cassandra, Prometheus, Kafka, Rsyslog, RabbitMQ and Elastic.

The Python Operator Framework supports Linux and Windows, on x86 and ARM.

Universal Operator Framework

Operators for legacy applications on VMs, clouds, and bare metal. YES.

Go beyond config management with the operator pattern - control your software with software.

That’s a hot new thing in Kubernetes, but it’s just as great for legacy estate. We also use operators for traditional applications on VMware or OpenStack, on public cloud, and on bare metal.

A universal framework connects new workloads on Kubernetes with old workloads on legacy infrastructure, transparently and seamlessly, because both are driven by consistent operators.

Integration Included

Application awareness with packaged integration

Application relationships are first class in the Python Operator Framework. Every operator is aware of the applications it can connect to, and the integration code is packaged in the operator.

Great operators manage much more than the application lifecycle. They understand that enterprise software is always integrated with other software. The best operators handle the application graph, with dynamic lines of integration that evolve over time, as you integrate more apps, because that’s what happens in real life.

About Operators

Software that runs software. What could go wrong?

Containers are amazing, with a lot of complexity and detail. It is hard to drive them correctly. The operator pattern is automation for speed and correctness.

Think of an operator as a software robot that knows everything the world’s best admins know about running a particular workload. It captures in code the best practices for installing, integrating, upgrading, scaling and assessing that workload. Using the operator is like hiring the world’s best specialist administrator for that workload. Free.

Learn more about operators, and read our Open Operator Manifesto, the critical factors for modern devsecops.

Beyond Configuration Management

Stop maintaining so much homegrown operations stuff

Traditional config management doesn’t capture knowledge in a general way. Each deployment is hardcoded. There is little efficient ops code reuse. You may be able to find snippets of code that are useful to cut-and-paste, but then you are maintaining everything.

Open operators fix that. They separate ops code from the deployment-specific model, so an open operator is completely reusable in other scenarios and other organisations.

Reuse community operators. If you need to create one, write it once - properly - and share it, then the maintenance burden is shared as well.
